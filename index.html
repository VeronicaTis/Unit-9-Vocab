Unit 9 Vocabulary
Web Form
  • Allows users to enter data that can be saved and processed
  • Common way to accepts user input
  • Allows the creation of interactive websites for users feedback
Controls are the objects that allow a user to interact with a form
Widgets are the objects that allow a user to interact with a form
Data Field stores the data values supplied by a user
Input boxes to insert any numeric values
Option/Radio buttons to select data values from a predefined set of options
Selection lists to select data values from an extensive list of options
Check boxes to select data values limited to two possibilities, such as “yes” or “no”
Text area boxes to enter text strings that may include several lines of content
Spin boxes to enter integer values confined to a specified range
Slider controls to enter numeric values confined to a specified range
Calendar controls to select date and time values
Color pickers to choose color values
Get method tells the browser to append the form data to the end of the url specified in the action attribute
Post method sends the form data in its own separate data stream. The post method is considered to be a more secure form of data transfer
Field set groups fields that share a common purpose
Virtual keyboards are software representations of a physical device
Spinner Control displays an up or down arrow to increase or decrease the field value by a set amount
Range Slider limits a numeric field to a range of possible values
Slide Control limits a numeric field to a range of possible values
Data list a list of possible data values that a form field can have
Form buttons a type of form control that performs an action
Command button runs a program that affects the content of a page or the actions of a browser
Submit button submits a form to the server for processing when clocked
Reset button resets a form, changing all field to their default values and deleting any field values that a user has entered
Validation process of ensuring that a user has supplied valid data
Server-side validation validation occurs on the web server(After you Submit)
Client-side validation validation occurs in the user’s browser(Before you Submit)
Regular expression a concise description of a character pattern
Regex a concise description of a character pattern
Inline validation the technique of immediate data validation and reporting of errors
Focus the state in which an element has been clicked by the user, making it the active control on the form

Attributes

Action attribute provides the location of the web server program that processes the form
Method attribute specifies how the browser should send form data to the server
Enctype attribute specifies how the form data should be encoded as it is sent to the server
Name provides the name of the data field associated with the control
id identifies the form
type indicates the data type of the field
for
Value is used to specify a default field value
Placeholder Text strings that appear within a form control, providing a hint about the kind of data that should be entered into a field.
rows specifies the number of lines in the text area box
cols specifies the number of characters per line
Multiple To allow more than one item to be selected
Size To display a selection list as a scroll box
label To associate a text string with a control, the text string has to be enclosed within the label element
Onclick
Required to identify the required data fields
pattern To validate a text value against a regular expression
maxlength maximum number of characters in the field value
focus to change the display style of fields that currently contain invalid data
